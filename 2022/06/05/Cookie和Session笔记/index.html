Cookie和Session笔记

发表于 2022-06-05

Cookie 基本使用

Cookie：客户端会话技术，将数据保存到客户端，以后每次请求都携带Cookie数据进行访问。

  1. 发送 Cookie
  • 查看浏览器的 Cookie

  1. 获取Cookie

Cookie 原理

Cookie的实现是基于HTTP协议的

  • 响应头：set-cookie
  • 请求头：cookie

①Servlet 将Cookie数据发送给浏览器，其实是通过响应头的方式。

②Servlet接收浏览器发送过来的Cookie数据，其实是通过请求头的方式。

Cookie存活时间

  • 默认情况下，Cookie存储在浏览器内存中，当浏览器关闭，内存释放，则Cookie被销毁。

  • setMaxAge（int seconds）：设置Cookie存活时间

    1. 正数：将Cookie写入浏览器所在电脑的硬盘，持久化存储。到时间后自动删除
    2. 负数：默认值，Cookie在当前浏览器内存中，当浏览器关闭，则Cookie被销毁
    3. 零：删除对应Cookie

Session 基本使用

  • Session：服务端会话跟踪技术，将数据保存到服务端。
  • JavaEE 提供HttpSession接口，来实现一次会话的多次请求间数据共享功能使用。

1、获取Session对象

  • HttpSession session = request.getSession（）；

2、Session对象功能：

  • void setAttribute（String name，Object o）：存储数据到 session 域中
  • Object getAttribute（String name）：根据key，获取值
  • void removeAttribute（String name）：根据 key，删除该键值对

Session原理

  • 一次会话，多次请求。session是如何共享数据的，其原理还是基于Session是基于Cookie实现的。第一次请求时，创建session对象时，以响应头的方式将数据JSESSIONID发送给客户端浏览器，浏览器将JSEESION以cookie的方式保存到浏览器中，下一次请求时，浏览器将JSESSIONID数据以请求头的方式发送给Servlet，这样就能获取到同一session，以达到共享数据。

Session钝化、活化

  • 服务器重启后，Session中的数据是否还在？
  • 钝化：在服务器正常关闭后，Tomcat会自动将Session数据写入硬盘的文件中
  • 活化，再次启动服务器后，从文件中加载数据到Session中

Session销毁

  • 默认情况下，无操作，30分钟自动销毁

  • 调用Session对象的invalidate（）方法

小结

Cookie 和 Session 都是来完成一次会话内多次请求间数据共享的

区别

  • 存储位置：Cookie是将数据存储在客户端，Session将数据存储在服务端
  • 安全性：Cookie不安全，Session安全
  • 数据大小：Cookie最大3KB，Session无大小限制
  • 存储时间：Cookie可以长期存储，Session默认30分钟服务器性能
  • Cookie不占服务器资源，Session占用服务器资源
